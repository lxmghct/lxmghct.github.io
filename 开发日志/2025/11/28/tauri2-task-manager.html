    Tauri 2 Task Manager 技术文档

    说明：本项目是一个使用 Tauri 2 + Vue 构建的本地个人日程 / 任务管理桌面应用，任务数据以按日 CSV 文件的形式存储在本地文件系统中。本文面向开发者，详细说明应用架构、目录结构、前后端交互、任务数据模型、悬浮球与倒计时等核心特性，便于后续维护与扩展。

    提示：文中提到的「截图」位置，可以在后续运行应用时自行抓图插入。

    1. 项目概览

    1.1 技术栈

    • 桌面壳层：Tauri 2
      • Rust 后端作为本地服务入口，暴露命令给前端
      • 使用 tauri.conf.json 配置多 Webview 窗口（主窗口、悬浮球、倒计时、菜单等）
    • 前端框架：Vue 3（<script setup> 语法）
    • 前端 UI / 功能：
      • 主任务管理界面：任务列表、按天管理、快捷操作
      • 浮动球（FloatBall）：全局小图标，支持拖拽、右键菜单、双击打开主窗口
      • 倒计时（Countdown）：圆形进度倒计时浮窗，支持拖动、暂停、停止等
      • 自定义右键菜单（CustomContextMenu）：在固定位置弹出的小窗体菜单
    • 前端与 Tauri API：
      • @tauri-apps/api/webviewWindow：多窗口管理
      • @tauri-apps/api/core：invoke 调用 Rust 命令
      • @tauri-apps/api/event：跨窗口事件通信
      • @tauri-apps/api/dpi：PhysicalPosition、PhysicalSize 控制窗口像素级位置和大小
    • 数据存储：本地文件（按日 CSV），配合后端 Task Service 读写

    [截图建议：应用主界面、浮动球、倒计时浮窗、右键菜单等整体效果图]

    3. 核心业务：任务与日程管理

    本节描述任务管理的核心设计。具体字段名称与类型以 src-tauri/src/task/model.rs 和 src/utils/task.js 为准。

    3.1 任务数据模型（Rust 后端）

    任务数据模型在 model.rs 中定义，一般会包含如下信息：

    • id：任务唯一标识（字符串或整数）
    • title：任务标题
    • description（可选）：任务描述
    • date：所属日期（如 2025-11-28），用来映射到对应的 CSV 文件
    • status：任务状态（未开始 / 进行中 / 已完成 等）
    • start_time / end_time（可选）：任务起止时间
    • priority（可选）：优先级

    [截图建议：代码中结构体 Task / TaskItem 的定义片段]

    这些字段通常会实现：

    • serde::Serialize / serde::Deserialize：便于与前端 JSON 通信
    • Clone / Debug 等 trait：便于调试与复制

    3.2 任务持久化：按日 CSV 文件

    service.rs 中实现的是任务读写逻辑，核心思路：

    1. 文件组织方式
      • 以日期为单位，每天一个 CSV 文件，例如：
        • data/2025-11-28.csv
        • data/2025-11-29.csv
      • 文件路径通常由基础目录（如 tasks_root_dir）+ 日期字符串拼接而成。
    2. 数据编码方式
      • 每行表示一个任务，采用 CSV 形式存储各字段。
      • service.rs 中会负责字段顺序的约定（例如：id,title,status,start_time,end_time,...）。
    3. 读取逻辑
      • 根据指定日期构造 CSV 文件路径；
      • 如果文件不存在，返回空任务列表；
      • 如果文件存在，逐行解析，每行映射到一个 Task 结构体；
      • 将解析后的 Task 列表返回给前端（序列化为 JSON）。
    4. 写入 / 更新逻辑
      • 接收前端提交的任务列表（或单个任务变更）；
      • 根据日期读取当前 CSV 文件内容，修改相应记录或追加新行；
      • 按照统一的字段顺序覆盖写回。

    [截图建议：service.rs 中读取 / 写入 CSV 的核心函数代码]

    3.3 Task 模块组织

    src-tauri/src/task/mod.rs 中负责：

    • 将内部的 model, service 等模块对外导出；
    • 向 lib.rs 暴露可供 Tauri 前端 invoke 的命令函数，例如：
      • #[tauri::command] fn get_tasks(date: String) -> Vec<Task>
      • #[tauri::command] fn save_task(task: Task)
      • #[tauri::command] fn delete_task(id: String, date: String)

    这些命令会在 lib.rs 中被注册。

    4. Tauri 后端入口：lib.rs

    具体代码以 src-tauri/src/lib.rs 为准。

    4.1 Tauri App 初始化

    lib.rs 典型结构：

    • tauri::Builder::default() 创建应用；
    • 注册命令： 
      tauri::Builder::default()
    .invoke_handler(tauri::generate_handler![
        get_tasks,
        save_task,
        delete_task,
        exit_app,
        // ... 其他命令
    ])
    .run(tauri::generate_context!())
    .expect(\"error while running tauri application\");
    • 如果需要，设置全局状态（例如任务根目录、配置路径等）。

    4.2 系统命令示例：退出应用

    在 FloatBall 中使用：

    import { invoke } from \"@tauri-apps/api/core\";

invoke(\"exit_app\");

    lib.rs 中对应：

    #[tauri::command]
fn exit_app(app_handle: tauri::AppHandle) {
    app_handle.exit(0);
}

    5. 前端任务工具：src/utils/task.js

    此文件负责前端对任务数据的辅助处理（如格式化、默认值、前端缓存等），并通过 invoke 与后端交互。

    常见职责包括：

    1. 创建默认任务对象
      • 提供函数 createEmptyTask(date) 或类似方法，快速生成指定日期的空任务对象。
    2. 格式化输入 / 输出
      • 将前端表单中的字段转换为后端要求的类型（例如日期字符串、时间字符串、状态枚举等）。
    3. 封装 Tauri 命令调用
      • 封装 getTasksByDate(date)、saveTask(task) 等函数，内部使用： 
        import { invoke } from \"@tauri-apps/api/core\";

export async function getTasksByDate(date) {
  return await invoke(\"get_tasks\", { date });
}
    4. 本地排序 / 过滤
      • 对任务列表按时间、优先级等排序；
      • 在前端做过滤（例如仅显示今日任务、仅显示未完成任务等）。

    6. 悬浮球与浮窗体系

    应用中比较有特色的一块是：悬浮球 + 动态浮窗，这部分主要由以下文件配合实现：

    • src/views/FloatBall.vue
    • src/views/Countdown.vue
    • src/views/CustomContextMenu.vue
    • src/utils/float-window.js
    • src/utils/window.js -（以及 Tauri 配置中的多窗口定义）

    6.1 FloatBall.vue：全局悬浮球

    核心模板：

    <template>
  <img
    id=\"ball-img\"
    src=\"../assets/float-ball.png\"
    draggable=\"false\"
    @mousedown=\"startDrag\"
    @mouseup=\"stopDrag\"
    @contextmenu.prevent=\"showMenu\"
    @dblclick=\"openMain\"
  />
</template>

    关键逻辑：

    • 通过 useFloatingWindow({ dragKey: \"float-ball\", getElement: ... }) 获取统一的拖拽与穿透逻辑；
    • @contextmenu.prevent=\"showMenu\"：右键点击时弹出自定义菜单（openMenu，内部通过新开菜单窗口 + 事件通信实现）；
    • @dblclick=\"openMain\"：双击悬浮球打开主任务窗口（openMainWindow）。

    右键菜单项示例：

    const menuItems = [
  {
    label: \"打开主窗口\",
    onClick: () => openMain(),
  },
  {
    label: \"<span style='color:#ff6666'>退出程序</span>\",
    onClick: () => invoke(\"exit_app\"),
  },
];

    6.2 Countdown.vue：倒计时浮窗

    Countdown.vue 是一个带圆形进度环的倒计时窗口，主要特性：

    • 使用 <canvas> 绘制背景圆环与进度环；
    • 输入分钟与秒数，点击「开始」后按秒数倒计时；
    • 支持暂停 / 继续、停止；
    • 倒计时结束后进度环在两种颜色之间闪烁提示；
    • 支持拖拽与屏幕穿透：通过 useFloatingWindow({ dragKey: \"countdown\", getElement: () => canvas.value }) 实现。

    核心状态：

    const inputTime = ref({ minutes: 0, seconds: 0 });
const state = ref({
  isRunning: false,
  isPaused: false,
  isFlashing: false,
  flashState: false,
});
const displayTime = ref(\"00 : 00\");

    关键函数：

    • formatTimeInput：
      • 清洗输入（仅保留数字，去掉前导 0）；
      • seconds >= 60 时自动进位到 minutes；
    • startCountdown：
      • 根据 inputTime 计算总秒数；
      • 初始化状态，启动 setInterval(updateCountdown, 1000)；
    • updateCountdown：
      • 每秒减少 remainingSeconds，更新任务显示与圆环绘制；
      • 到 0 时停止计时并启动闪烁模式；
    • pauseCountdown / stopCountdown：
      • 暂停 / 恢复：通过控制 setInterval 的创建和清除；
      • 停止：清除所有定时器，状态归零，重绘空圆环；
    • startFlashing / stopFlashing：
      • 使用单独的 setInterval 切换 flashState 来实现闪烁动画。

    绘制逻辑（简化概念）：

    const drawCircle = () => {
  const ctx = canvas.value.getContext(\"2d\");
  ctx.clearRect(0, 0, width, height);

  // 1. 绘制灰色背景圆
  // 2. 如果在倒计时：按剩余时间计算进度，绘制彩色圆弧
  //    - 进度从红到绿渐变
  //    - 暂停时使用固定颜色（如黄色）
  // 3. 如果在闪烁：不按进度绘制，而是整圈在两种颜色之间切换
};

    返回主窗口：

    const back = () => {
  win.hide();
  setVisible(false);
  openMainWindow();
};

    6.3 CustomContextMenu.vue：自定义右键菜单窗口

    此组件负责在某个屏幕位置显示一个小菜单窗口，页内结构相对简单：

    • 使用 URL 查询参数接收数据：
      • windowLabel：需要回发事件的窗口标签；
      • data：菜单项数组（字符串列表），例如 [{ label: \"打开主窗口\" }, { label: \"退出\" }]；
    • 组件挂载时：
      1. 解析参数并填充 items；
      2. 通过 nextTick 计算实际 DOM 宽高；
      3. 调整窗口位置（防止超出屏幕边界）；
      4. 使用 PhysicalPosition / PhysicalSize 设置 Tauri 子窗口的位置和尺寸；
      5. 最后调用 win.show() 与 win.setFocus()。

    点击菜单项时：

    const handleClick = (i) => {
  if (windowLabel) {
    emitTo(windowLabel, \"context-menu-item-click\", { index: i });
  } else {
    emit(\"context-menu-item-click\", { index: i });
  }
  WebviewWindow.getCurrent().close();
};
    • 通过事件总线将所选菜单项的 index 返回给调用者窗口；
    • 关闭菜单窗口自身；
    • 页面失焦（blur 事件）时，也会触发 handleClick(-1)，代表用户取消选择。
